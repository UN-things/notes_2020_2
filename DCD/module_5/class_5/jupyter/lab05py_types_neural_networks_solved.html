Cargar el set de datos¶

Dependiendo del problema que queramos solucionar, va a ser necesario recolectar una gran cantidad de imgenes para entrenar el modelo. Existen fuentes de datos abiertas para casos de uso generales como: personas, animales, flores, automviles, etc.
Por otro lado, cundo nuestro problema es mucho ms particular y no se encuentran datasets ya creados para nuestro caso, ser necesario recolectar las imgenes manualmente. La regla siempre ser: Entre ms imgenes, mejor.
En cualquiera de los dos casos, la estructura recomendable para organizar las imgenes es la siguiente:

Directorio Raz
train
   clase1
          imagen01.jpg
          imagen02.jpg
          ...
   clase2
          imagen01.jpg
          imagen02.jpg
          ...
   ...
   
test
    clase1
           imagen01.jpg
           imagen02.jpg
           ...
    clase2
           imagen01.jpg
           imagen02.jpg
           ...
    ...

Los directorios no necesariemante deben llamarse as, pero s es recomendable tener un set de datos de entrenamiento separado del de pruebas. Adems, los nombres de las carpetas dentro de estos (clase1, clase2, ...), son los que indicarn las clases/labels de las imagenes que estn adentro.

Data Augmentation¶

Como ya se ha mencionado varias veces, entre ms datos tengamos, mejor. En el caso de Computer Vision: entre ms imgenes, mejor. Cuando no tenemos suficientes datos, la tcnica de Data Augmentation permite, por medio de un algoritmo, generar imgenes artificialmente a partir de las pocas imgenes que tenemos. Esto se logra haciendo cambios pequeos a los datos: Girar un poco la imagen, cambiando su luminocidad, resolucin, movindola un poco, etc. Para nosotros es un cambio pequeo, pero para el modelo es una imagen completamente diferente!
A nivel de cdigo, estas son TODAS las opciones que se pueden configurar para el generador de datos, tenga en cuenta que no siempre todas aplican, por ejemplo, la imagen del len mirando a la derecha o mirando a la izquierda, sigue siendo un len, pero, nuestros logos?

5b. Transfer Learning¶

Para no tener que disear la red desde ceros, pensando qu capas, neuronas y funciones de activacin usar, usamos Transfer Learning, en el que tomamos la estructura de una red ya contruida y especializada en una tarea especfica (Ej: Reconocimiento de imagen).
Los siguientes pasos iniciales siguen siendo necesarios:

  • Definicin de hiperprmetros
  • Generador de imagenes nuevas (Data Augemtnation)
  • Configuracin de generadores para entrenamiento y pruebas

Pero en lugar de crear un modelo desde ceros, vamos a tomar un modelo ya existente, por ejemplo VGG16 y vamos a indicar que sus capas NO SON reentrenables:
