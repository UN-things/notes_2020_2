Recordemos que una Red Neuronal es, al fin y al cabo, una ecuacin. Vamos a entrenar una Red Neuronal/Ecuacin sencilla, de la cul ya conocemos los coeficientes para poder validar su funcionamiento ya que estamos aprendiendo, por ejemplo: Transformar Grados Centgrados a Grados Farenheit.
Obviamente, podemos recordar o buscar la ecuacin y crear una funcin tradicional que haga el clculo:

Por el contrario, vamos a crear un modelo basado en algunos valores de grados Celsius, para los que conocemos sus correspondientes valores Farenheit y vamos a dejar que el modelo encuentre la relacin entre ellos.

Recordemos los tres pasos para implementar una Red Neuronal que vimos en clase:

1. Definir el Modelo
El modelo se define como un listado de capas a las cuales debemos configurar para cada una, el nmero de neuronas y la funcin de activacin. Por ahora, para empezar, vamos a definir solo una capa, Densa, con una sola Neurona:

2. Compilar el Modelo
Ahora debemos decirle al Modelo, cmo medir si se est equivocando (Funcin de prdida), cmo evaluarse a s mismo (Mtricas) y cmo ajustar sus valores (Optimizador) para ir mejorando en cada poca:

In [4]:
model %>% compile(
  loss = "mean_squared_error",
  optimizer = optimizer_adam(lr = 0.1),
  metrics = list("mean_absolute_error")
)

3. Entrenar el Modelo
Listo! Ya le indicamos a la red su estructura, y sus mtodos de aprendizaje, ahora solo falta indicarle con qu datos se va a entrenar y por cuanto tiempo lo va a hacer (pocas):

In [5]:
history <- model %>% fit(
  c, f, 
  epochs = 500,
  steps_per_epoch=12
)

Estas 500 pocas se ejecutaron muy rpido! Cuando estemos trabajando con un dataset real/ms grande (Con ms atributos y muchsimos ms registros) es posible que tome mucho ms tiempo, por esto es importante tener en cuenta la cantidad de pocas y pasos en cada poca.

En este ejemplo, vemos como, poca a poca, el modelo va evolucionando: Disminuye el valor de prdida (loss) y aumentan/disminuyen las mtricas elegidas (En este caso, Error Medio Absoluto, disminuye!).
Es mucho mejor ver esta evolucin del modelo grficamente para tomar decisiones. Si ejecutamos la sentencia anterior en una consola de R-Studio, nos va a mostrar la frfica de entrenamiento en tiempo real, pero si no, podemos generar la grfica final de la siguiente manera:

Aqu vemos de forma grfica cmo, en su primer intento (epoca), el modelo se equivoc muchsimo "adivinando" los coeficientes de la ecuacin, es decir el valor del loss dio muy alto al elegir los coeficientes aleatoriamente. Sin embargo, en las siguientes pocas, estos valores se fueron optimizando, disminuyendo el loss a casi 0.

Ejercicio:
Compile y entrene nuevamente el modelo (compile() y fit()) sin cambiar ningn parmetro. Genere nuevamente la grfica de prdida. Qu nota?

Es muy posible que la grfica sea diferente cada vez (Es decir: Las Redes Neuronales son No Determinsticas). Esto se debe a:

  • Los valores iniciales se eligen aleatoriamente y son distintos cada vez que se compila/entrena el modelo.
  • Lo mismo pasa con cada poca, para modificar los coeficientes de una poca a otra usando la funcin de optimizacin se toman valores aleatorios muy pequeos (Taza de Aprendizaje, LR).
  • En todo caso, aunque se trata de varias decisiones aleatorias, las "reglas del juego": optimizador, mtricas y funcion deprdida, ayudan al modelo a encaminarse a una buena solucin

Taller:

Qu pasa si aadimos ms capas y neuronas a la Red? O si cambiamos el mtodo de aprendizaje?

  • En la definicin del modelo, puede agregar cualquiera o varias de las siguientes capas a la funcin, en cualquier orden y modificar los valores x de sus hiperparmetros:

model <- keras_model_sequential()
model %>% layer_dense(x, input_shape=c(1)) # No cambiar input_shape (#entradas este caso = 1)
model %>% layer_dense(x) # A partir de la 2 capa podemos "jugar" con la cantidad de neuronas
model %>% layer_dropout(x) # Tambin podemos modificar otrss parmetros de cada una
model %>% layer_dense(1) # No cambiar (#salidas este caso = 1)

  • En las capas principales puede cambiar tambin la funcin de activacin por los valores: activation = "relu", "softmax", "sigmoid", "tanh"
  • En la compilacin del modelo, intente con otros optimizadores, por ejemplo: optimizer_sgd, optimizer_adadelta, optimizer_adam, optimizer_rmsprop
  • En el entrenamiento del modelo, pruebe modificando los valores de epochs y steps_per_epoch

Tenga en cuenta que si va a trabajar un problema de Clasificacin, tambin puede "jugar" con loss = "categorical_crossentropy" y metrics = list("accuracy", "recall")...

Analice cmo cambia la estructura de la red (summary()), y como evoluciona en entrenamiento de acuerdo a su nueva configuracin (Mejora? Empeora? Aprende ms rpido?)
