OLS (Ordinary Least Squares)¶

Llamado as porque busca los coeficientes de la ecuacin que minimizen la suma de las distancias verticales entre los datos y el modelo. Para aplicar este modelo se debe validar que no haya multicolinealidad entre las variables predictivas, pues no la detecta fcilmente.

  • Precisin: Alta, para variables no correlacionadas (Condiciones para OLS aqu)
  • Velocidad: Rpido
  • Explicativo: Mucho (Peso/Importancia de cada variable)
  • Sensible a cambios: Poco, OLS crea una estimacin generalizada a todos los puntos, un solo valor atpico modificar los pesos de las variables pero no drasticamente, pues seguir convergiendo a los dems puntos
  • Deteminstico: S

En donde Y es nuestra variale objetivo y cada Xi son las variables predictivas para las que se debe buscar un peso en la ecuacin.

La funcion fit.summary() nos entrega un montn de informacin, vamos a analizar las que mas nos interesan:

  • Modelo utilizado (OLS), cantidad de registros y de variables usadas (incluyendo intercepto)
  • Df Residuals (Flexibilidad del modelo: #Observacionnes - #Variables) y Df modelo (#Variables -Intercepto) (Un dato que no da el summary de R)
  • R2, en la primera tabla, es el cuadrado del coeficiente de correlacin de todas las variables y representa el porcentaje estimado de varianza que puede ser explicado por el modelo.
  • Adj. R2 es un R2 que penaliza coeficientes muy grandes o muy bajos y variables redundantes.
  • La tabla del centro nos muestra los coeficientes encontrados por el modelo para cada variable, junto con el error estndar de cada uno. Esto significa que nuestro modelo tiene la forma:
$$ y = 153.5215 + 0.1088(fixed acidity) - 1.6731(volatile acidity) + ... + 0.2012(alcohol) $$
  • Los valores t y valores P (P>t), para cada variable, indican si esta afecta o no la variable objetivo. Son valores estadsticos, por lo general se busca que P < 0.05
  • Intervalo de confianza de cada variable [0.025 - 0.975]

Regresion Logística¶

Similar a una regresin lineal, busca definir el comportamiento de los datos a travs de una ecuacin, con la diferencia que en este caso estamos buscando una variable categrica (1/0 o SI/NO), por lo que no nos servira una ecuacin que retorne un valor entre $(-\infty, \infty)$.
Sin embago, al ser una ecuacin, asume que los datos son linealmente separables:

  • Precisin: Alta, para variables no correlacionadas y modelos linealmente separables
  • Velocidad: Rpido
  • Explicativo: Mucho (Peso/Importancia de cada variable)
  • Sensible a cambios: Mucho, asume que los datos son separables linealmente
  • Deteminstico: No, probabilstico

Como el hecho de que sean datos linealmente separables es muy importante, vamos a crear un modelo con solo dos variables precitivas para poder ver la separacin grficamente:

Vecinos más cercanos (kNN)¶

Es un mtodo de clasificacin para problemas no lineales, y tal vez con una metodologa/matemtica ms sencilla. Consiste en ver los datos como puntos en un plano (de N dimensiones cuntas variables incluya el modelo) y clasificar cualquier "punto nuevo" en la misma clase a la que pertenescan los k puntos ms cercanos (k puede ser definido por nosotros mismos)

  • Precisin: Depende intensamente del valor de k. muy eficiente para datasets sin ruido
  • Velocidad: Depende de dimensiones (Realmente no construye ningn modelo)
  • Explicativo: Mucho (Es una ecuacin sencilla)
  • Sensible a cambios: Mucho a atributos irrelevantes y la complejidad del modelo
  • Deteminstico: S

En este caso, se eligi el nmero de vecinos (k) aleatoriamente, y con razn no obtuvimos un score bueno. Pero cuidado:

  • Si k es muy pequeo, tendiendo a 1, el modelo va a decidir por la clase del ejemplo ms cercano. Esto hace el modelo muy sensible al ruido o a pequeos cambios.
  • Si k es muy grande, tendiendo a N, el modelo no podr gneralizar. Nunca analizar patrones, reglas, ni similitudes sino que decidir siempre lo que diga la mayora

Máquinas de Soporte Vectorial (SVM)¶

Cuando hablamos de Regresin logstica, buscamos una lnea que divida los datos de la mejor manera. Pero, qu pasa cuando hay ms de una lnea que cumple con esta condicin? Cmo elegir la mejor?
Las maquinas de soporte vectorial buscan, por el contrario, un hiperplano: un conjunto de lneas que maximicen la distancia/margen entre los puntos ms cercanos de cada clase. A estos puntos ms cercanos se les conoce como vectores de soporte

  • Precisin: Alta si se configura correctamente
  • Velocidad: Lento para una gran cantidad de datos
  • Explicativo: No
  • Sensible a cambios: Si, sobretodo a ejemplos mal etiquetados
  • Deteminstico: S

Árboles de Decisión¶

Es el tipo de algoritmo ms fcil de entender, pues se basa en reglas condicionales muy similares al lenguaje humano (Si... entonces...). El entrenamiento consiste en encontrar las reglas y condiciones que cubran la mayor cantidad de casos.
El modelo grficamente puede ser representado mediante un rbol en el que los nodos intermedios son decisiones basadas en alguno de los atributos predictivos, y las hojas son la clase objetivo.

  • Precisin: Alta tendencia al sobreajuste (memorizar datos)
  • Velocidad: Rpido. La validacin de reglas son condicionales
  • Explicativo: S
  • Sensible a cambios: Si, pequeos cambios en los datos puede resultar en rboles completamente diferentes.
  • Deteminstico: No
In [31]:
x_train, x_test, y_train, y_test = train_test_split(x, y, test_size=0.2, random_state=42)
In [32]:
model_tree= DecisionTreeClassifier(max_depth=3)
model_tree.fit(x_train, y_train)
print('Error: {}'.format(1 - model_tree.score(x_test, y_test)))

Error: 0.3015873015873016

En cada uno de los nodos, el algoritmo valida qu variable y qu valor de esa variable produce un nivel mayor de "pureza" entre las hojas (Entropa/gini index). Si este nivel no es suficiente para dar una buena decisin, se hace una nueva particin en cada una de las hojas, validando nuevamente todas las variables predictivas.
Por esto mismo, los rboles de decisin tienen varios parmetros que configurar (Nivel mnimo de entropa, profundidad mxima del rbol, mnima cantidad de muestras para las hojas, etc.)
Otra de las ventajas de los rboles es que son tan explicativos, que pueden ser graficados para validar el proceso de toma de cada decisin en el proceso de prediccin:

Ensambles de Modelos¶

Cuando un modelo es no determinstico, pero tiende al sobreajuste, es decir, aprende de memoria las reglas y patrones del set de entrenamiento, hasta el punto de no predecir correctamente sobre el test de entrenamiento, esto puede solucionarse haciendo un ensamble o conjunto de modelos. Esto es, construir varios modelos "dbiles" y crear uno "robusto", tomando la decisin final dependiendo de:

  • "Democracia", el resultado es lo que diga la mayora de modelos, realizando cada uno bajo un set de entrenamiento diferente (BaggingClassifier)
  • "Aleatorio", el resultado es lo que diga la mayora de rboles de decisin, realizando cada uno con un set aleatorio de variables (RandomForestClassifier)

  • Precisin: Alta, varios modelos eliminan el sobreajuste
  • Velocidad: Lento, por velocidad se sacrifica precisin
  • Explicativo: S, aunque no entrega el modelo, entre importancia de variables
  • Sensible a cambios: No mucho, un dato atpico peude modificar un rbol, pero no todos
  • Deteminstico: No

x_train, x_test, y_train, y_test = train_test_split(x, y, test_size=0.2, random_state=42)
In [36]:

El concenso realizado entre los modelos permite al algoritmo, a pesar de ser no determinstico, darse una idea de la importancia de cada variable en la toma de la decisin:
